# Rule Engine

TalkBridge 규칙 엔진은 외부 API 없이 문자열 규칙을 순서대로 적용합니다.

`provider`를 `local-gec` 또는 `hybrid`로 지정하면 Python worker가 로컬 Hugging Face GEC 모델을 먼저 실행합니다. `hybrid`는 모델 결과를 다시 규칙 엔진으로 후처리합니다.

## 처리 순서

1. URL과 이메일을 보호 토큰으로 치환합니다.
2. 앞뒤 공백, 중복 공백, 문장부호 주변 공백을 정리합니다.
3. 흔한 오타를 고칩니다.
4. 한국어 채팅에서 자주 붙여 쓰는 표현을 띄어 씁니다.
5. 선택한 말투에 따라 정중한 표현, 친근한 표현, 간결한 표현으로 다듬습니다.
6. 인사 뒤 마침표와 문장 끝 문장부호를 보완합니다.
7. 보호 토큰을 원래 URL과 이메일로 되돌립니다.

## 포함 규칙 예시

- `안녕하세요 잘지내셨나요` -> `안녕하세요. 잘 지내셨나요?`
- `보내기전에` -> `보내기 전에`
- `확인부탁` -> `확인 부탁`
- `안되요` -> `안 돼요`
- `몇일` -> `며칠`
- `오랫만` -> `오랜만`
- `자료 확인부탁` + manager -> `자료 확인 부탁드립니다.`

## 보존 정책

URL, 이메일, 숫자, 이모지는 직접 교정 대상이 아닙니다. URL과 이메일은 보호 토큰으로 감싼 뒤 마지막에 복원합니다.

## 한계

규칙 기반 엔진만으로는 문맥 의존적인 교정, 복합 의미 판별, 대화 상대에 맞춘 섬세한 톤 조절에 한계가 있습니다. 더 자유로운 입력을 처리하려면 `local-gec` 또는 `hybrid` provider를 사용합니다.
